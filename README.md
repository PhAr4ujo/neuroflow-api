# Neuroflow API

Laravel API application with OpenAPI documentation generated by Scramble and rendered with Scalar.

## Requirements

- PHP 8.3 or newer
- Composer
- Node.js and npm
- MySQL 8.4 or a compatible MySQL server
- Docker, if you prefer running the database and app through Laravel Sail

## Run Locally

1. Install PHP dependencies:

```bash
composer install
```

2. Create the environment file and application key:

```bash
cp .env.example .env
php artisan key:generate
```

3. Configure the database in `.env`.

For a local MySQL server, the defaults are:

```dotenv
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=neuroflow_api
DB_USERNAME=root
DB_PASSWORD=
```

If you use the MySQL service from `compose.yaml` while running Artisan on your host machine, use:

```dotenv
DB_HOST=127.0.0.1
DB_PORT=3307
```

4. Run the database migrations:

```bash
php artisan migrate
```

5. Install frontend dependencies:

```bash
npm install
```

6. Start the application:

```bash
composer run dev
```

This starts the Laravel development server, queue listener, log viewer, and Vite development server together.

The API will be available at `http://127.0.0.1:8000`.

## Run With Laravel Sail

1. Install PHP dependencies and create the environment file:

```bash
composer install
cp .env.example .env
```

2. Update the database host in `.env` for Docker networking:

```dotenv
DB_HOST=mysql
DB_PORT=3306
```

3. Start the containers:

```bash
./vendor/bin/sail up -d
```

4. Install dependencies, generate the key, and run migrations inside Sail:

```bash
./vendor/bin/sail composer install
./vendor/bin/sail artisan key:generate
./vendor/bin/sail artisan migrate
./vendor/bin/sail npm install
./vendor/bin/sail npm run dev
```

By default, Sail exposes the app at `http://localhost` and MySQL on host port `3307`.


## API Documentation

This project uses Scramble to generate the OpenAPI document and Scalar to render the API reference UI.

- Scalar UI: `/docs/api`
- OpenAPI JSON: `/docs/openapi.json`

Notes:

- The OpenAPI document generation reads the MySQL schema.
- If you run the app through Sail, the current `.env` database host (`mysql`) is correct.
- If you run the app directly on the host machine, use the host-mapped MySQL connection instead, which is `127.0.0.1:3307` in the current `compose.yaml`.
